# microsoft_fabric_olist_saas_project
Microsoft Fabric Olist SaaS Project: End-to-End E-Commerce Analytics📌 Project OverviewThis project demonstrates an enterprise-grade data engineering and analytics pipeline built entirely within Microsoft Fabric. It ingests raw e-commerce data (Olist dataset) and processes it through a Medallion Architecture (Bronze, Silver, Gold) using PySpark. The final output is an optimized Star Schema connected to Power BI via zero-latency DirectLake mode.🏗️ Architecture & TechnologiesPlatform: Microsoft Fabric (OneLake, Lakehouse, SQL Analytics Endpoint)Data Engineering: PySpark, Apache Spark, Delta Lake formatData Modeling: Dimensional Modeling (Star Schema)Business Intelligence: Power BI (DirectLake connectivity), DAX⚙️ Data Pipeline Execution (The Medallion Architecture)🥉 Bronze Layer: Raw Data IngestionFile: 01_Ingest_Raw_to_Bronze.ipynbIngested raw Olist E-commerce CSV files into OneLake.Maintained raw fidelity for historical auditing and lineage.🥈 Silver Layer: Cleansing & ConformingFile: 02_Transform_Silver_Layer.ipynbPySpark Transformations: Applied schema enforcement, standardized data types, and cast string dates to proper timestamps.Data Quality: Handled missing values and standardized naming conventions (e.g., silver_customers).Storage: Saved cleaned datasets as optimized Delta Parquet tables.🥇 Gold Layer: Semantic Modeling (Star Schema)File: 03_Gold_Dimensional_Model final.ipynbDimensional Modeling: Engineered a robust Star Schema optimized for downstream analytical performance.Entity Resolution: Handled complex primary key relationships, such as resolving the customer_unique_id vs customer_id grain trap to eliminate dimensional duplicates.Final Assets: Produced clean, aggregated Delta tables: fact_sales, dim_customers, dim_products, and dim_sellers.📊 Visual Proof & BI IntegrationThe pipeline orchestration and final dashboard are documented below:End-to-End Process FlowFinal Data PipelineDirectLake Mode: Leveraged native connectivity to eliminate data import latency, allowing Power BI to query Delta files directly from OneLake.Business Logic: Authored DAX measures to calculate core business KPIs:Total Revenue: $\sum (\text{fact\_sales}[\text{SalesAmount}])$Total Orders: $\text{COUNT}(\text{fact\_sales}[\text{OrderKey}])$
